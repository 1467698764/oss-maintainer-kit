# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a vulnerability. Contact the maintainer privately with:

- affected version or commit,
- reproduction steps,
- impact and expected behavior,
- any suggested mitigation.

## Supported versions

The latest release line receives security fixes. Because this project runs locally and does not process untrusted network input by default, most issues are expected to be local file handling or packaging concerns.
